本文介绍了本站所使用的jekyll配置和技术细节。  

Jekyll将文件作为数据库加载到内存中，结构化后提供给liquid作为接口进行渲染。  
```ruby
\{\{ \}\}                   # 放置变量
\{\{ object | filter \}\}   # https://jekyllrb.com/docs/liquid/filters/
\{\% \%\}                   # liquid 逻辑语句
```

在被视为有效的数据源中，html或者markdown请添加**frontmatter(yaml前言)**声明了其基本配置和额外自定义数据。  
有一些默认的键比较重要，通常是布局，标题，分类，标签等。  
具体的所有可用数据在下方解释。  
```ruby
\---
\my_number: 5
\---
\{\{ page.my_number }} # 被解析到所谓的page数据中
```

## 帖子和页面
**帖子**的数据库完全基于文件夹(_posts)被识别到变量封存中。再由主页和模板进行导航。  
```ruby
# _posts
#   a.md
```
帖子给定的前言这里的指定布局是一个子布局文件。该子布局本身必须在其前言中指定一个基础布局(通常是Html的根部结构)。  
在子布局中*\{\{ content }}`*将替换为帖子内容(转为html)，而其前言中指定基础布局中的*\{\{content}}*将替换为子布局的渲染结果。  
实现了post的一致性。  
```ruby
# post
\---
layout: sublayout
# title:  ""
# author:
# category:
# tags: 
\---

# sublayout
\---
layout: rootlayout
\---
```

普通的html页面(如index.html)通常与帖子区分，对于本站而言，无需单独为他们设置目录。  
如前所述，帖子具有专属文件夹，我们固定方式管理，使用标签和子文件夹进行分类。  
为它们设置前言，如前文所述。  
收藏是另一个jekyll的分组功能，本站不使用。  
数据文件可以视为jekyll站点中的小型数据库。
