本文介绍了本站所使用的jekyll配置和技术细节。  

Jekyll将文件作为数据库加载到内存中，结构化后提供给liquid接口以便其进行渲染。  
```ruby
\{\{ \}\}                   # 输出变量
\{\{ object | filter \}\}   # https://jekyllrb.com/docs/liquid/filters/
\{\% \%\}                   # liquid 逻辑语句
```

在被视为有效的数据源中，你喜欢的标记语言html或者markdown请添加**frontmatter(yaml前言)**声明了其基本配置和额外自定义数据。  
有一些默认的键比较重要，通常是布局，标题，分类，标签等。  
具体的所有可用数据在下方解释。  
```ruby
\---
\my_number: 5
\---
\{\{ page.my_number }} # 被解析到所谓的page数据中
```
如果文本包含liquid，则必须包含前言结构。  

## 帖子和页面
**帖子**的数据库完全基于文件夹(_posts)被识别到变量封存中。再由主页和模板进行导航。  
```ruby
# _posts
#   a.md
```
帖子给定的前言这里的指定布局允许是一个子布局文件。该子布局本身允许在其前言中指定一个嵌套基础布局(通常是Html的根部结构)。  
在子布局中*\{\{ content }}`*将替换为帖子内容(转为html)，而其前言中指定基础布局中的*\{\{content}}*将替换为子布局的渲染结果。  
实现了post的一致性。  
```ruby
# post
\---
layout: sublayout
# title:  ""
# author:
# category:
# tags: 
\---

# sublayout
\---
layout: rootlayout
\---
```

普通的html页面(如index.html)通常与帖子区分，对于本站而言，无需单独为他们设置目录，并且只使用html格式。  
如前所述，帖子具有专属文件夹，我们通过固定方式管理，使用标签和子文件夹进行分类。  
为它们设置前言，如前文所述。  
集合(collections)是另一个jekyll的分组功能，本站不使用。首先需要在配置中声明集合。  
数据文件被用作jekyll站点中的小型数据库。然后类似于帖子数据，通过顶级数据进行访问。  

## 数据
一个重要参考是jekyll的数据结构。请见[連結](https://jekyllrb.com/docs/variables/)。  
其他不太重要的jekyll参考，仅在特定方面开发时可能需要参考确认，[请从该完整列表查看](https://jekyllrb.com/docs/pages/)。本文已涵盖大多数核心。  
```ruby
site ->    # 全站的遍历数据
page ->    # 当前页面/帖子的数据
jekyll ->  # jekyll信息
paginator -> # 分页器支持的数据
content    # 在子内容渲染完成后，待插入父级的内容
```

## 主题等
jekyll允许直接使用设计好的主题、插件、额外的Gem代码来添加功能。  
本站未使用任何主题、插件、以及Gem。  

尽管Jekyl允许分页，本站仅使用includes模板生成的上下单页链接进行分页和导航。
